# 永辉大模型认知与使用情况调查问卷系统 — 需求文档

## 1. 项目概述

### 1.1 项目背景

为了解永辉内部员工对现有大模型（如 ChatGPT、Claude、文心一言、通义千问等）的认知程度和实际使用情况，需要搭建一套在线问卷调查系统，用于收集员工反馈并生成可视化统计报告。

### 1.2 项目目标

- 提供一个简洁易用的问卷填写界面，支持选择题、判断题、简答题
- 填写者需登录后作答，系统记录每位填写者的身份信息
- 管理员可在后台创建/编辑问卷题目，设置问卷截止时间
- 系统防止同一用户重复提交
- 管理端可查看统计图表（选择题选项分布、填写进度等）

---

## 2. 用户角色

| 角色 | 说明 |
|------|------|
| **填写者（普通用户）** | 永辉员工，登录后填写问卷 |
| **管理员** | 问卷的创建者/管理者，可编辑题目、查看统计结果 |

---

## 3. 功能需求

### 3.1 用户端（问卷填写）

- **登录认证**：用户通过账号密码登录（或工号登录），系统记录填写者身份
- **问卷列表**：登录后展示当前可填写的问卷（未过截止时间 + 未提交过的）
- **问卷填写页**：
  - 支持三种题型：单选题、多选题、判断题（是/否）、简答题
  - 题目按顺序展示，支持上下滚动
  - 填写过程中可暂存（可选功能）
- **提交**：
  - 点击提交后数据入库
  - 防止重复提交：已提交的问卷标记为"已完成"，不可再次提交
  - 提交后展示"感谢参与"页面
- **截止时间**：问卷超过截止时间后，不再允许填写

### 3.2 管理端（后台管理）

- **问卷管理**：
  - 创建新问卷（填写标题、描述、截止时间）
  - 编辑已有问卷的题目
  - 删除问卷
- **题目管理**：
  - 新增题目，选择题型（单选 / 多选 / 判断 / 简答）
  - 对于选择题和判断题，设置选项内容
  - 调整题目顺序
  - 编辑和删除题目
- **数据统计与结果查看**：
  - 查看问卷的填写人数 / 总人数（填写进度）
  - 选择题 / 判断题：以柱状图或饼图展示各选项的选择分布
  - 简答题：以列表形式展示所有回答
  - 支持按部门等维度筛选
- **用户管理**（基础）：
  - 管理员可查看已注册用户列表
  - 可手动添加用户或批量导入

## 4. 数据模型设计（参考）

### 4.1 用户表 (users)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT / UUID | 主键 |
| username | VARCHAR | 用户名 / 工号 |
| password | VARCHAR | 加密密码 |
| name | VARCHAR | 真实姓名 |
| department | VARCHAR | 部门 |
| role | ENUM | user / admin |
| created_at | TIMESTAMP | 创建时间 |

### 4.2 问卷表 (surveys)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT / UUID | 主键 |
| title | VARCHAR | 问卷标题 |
| description | TEXT | 问卷描述 |
| deadline | TIMESTAMP | 截止时间 |
| status | ENUM | draft / published / closed |
| created_by | FK → users.id | 创建者 |
| created_at | TIMESTAMP | 创建时间 |

### 4.3 题目表 (questions)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT / UUID | 主键 |
| survey_id | FK → surveys.id | 所属问卷 |
| type | ENUM | single_choice / multi_choice / true_false / short_answer |
| content | TEXT | 题目内容 |
| options | JSON | 选项数组（选择题/判断题用） |
| order | INT | 题目排序 |
| required | BOOLEAN | 是否必答 |

### 4.4 回答表 (responses)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT / UUID | 主键 |
| survey_id | FK → surveys.id | 所属问卷 |
| user_id | FK → users.id | 填写者 |
| submitted_at | TIMESTAMP | 提交时间 |

### 4.5 答案明细表 (answers)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT / UUID | 主键 |
| response_id | FK → responses.id | 所属提交记录 |
| question_id | FK → questions.id | 所属题目 |
| answer_value | TEXT | 用户的回答内容 |

## 5. 技术栈建议

### 方案 A：轻量级全栈（推荐，适合快速开发）

- **前端**：React + Ant Design（或 Next.js）
- **后端**：Node.js + Express / Nest.js
- **数据库**：PostgreSQL 或 MySQL
- **图表**：ECharts 或 Recharts
- **认证**：JWT Token

### 方案 B：Python 全栈

- **前端**：React + Ant Design
- **后端**：Python + FastAPI
- **数据库**：PostgreSQL
- **图表**：ECharts
- **认证**：JWT Token

---

## 6. 页面路由规划

```
用户端：
/login                    → 登录页
/surveys                  → 问卷列表
/survey/:id               → 问卷填写页
/survey/:id/complete      → 提交成功页

管理端：
/admin/login              → 管理员登录
/admin/dashboard          → 数据概览（填写进度）
/admin/surveys            → 问卷列表管理
/admin/survey/:id/edit    → 编辑问卷题目
/admin/survey/:id/stats   → 查看统计结果
/admin/users              → 用户管理
```

---

## 7. 非功能需求

- **安全性**：密码加密存储，接口鉴权，管理端与用户端权限隔离
- **响应式**：支持 PC 和移动端浏览器访问
- **性能**：支持至少 500 人同时在线填写
- **数据备份**：数据库定期备份
